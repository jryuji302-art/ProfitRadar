# App Version Decision

## 結論

正本は app.py とする。

## 理由

app.py:

- ROIあり
- AI分析あり
- Gmail接続あり
- 返信検知あり
- マルチユーザー対応あり
- 現在の本番に近い可能性が高い

app_next.py:

- UIタブ名は理想に近い
- 今すぐ返信タブあり
- 開発者タブあり
- 顧客/実績の見え方は整理されている
- ただしROIが未検出

## 方針

app_next.pyをそのままapp.pyへ昇格しない。

app.pyを正本にして、app_next.pyの良いUXだけ取り込む。

## 取り込む候補

- タブ名：
  - 🏠 今日の利益
  - 🔥 今すぐ返信
  - 👥 顧客
  - 📈 実績
  - ⚙️ Gmail接続

- 開発者タブ：
  - dev_mode時のみ表示

- 返信あり：
  - 独立タブではなく、今日の利益または今すぐ返信に統合

## 最優先修正

1. 利用者画面の RUDIA / ルディア 表記除去
2. 旧 要対応タブ残骸削除
3. タブ名整理
4. Gmail OAuth 固定 user_id/company_id 修正
5. UI分割準備
