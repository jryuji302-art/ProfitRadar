# Profit Radar UI Blueprint

## 目的

Profit RadarはCRMではなく、AI営業参謀。

利用者が見るべきものは以下だけ。

1. 今日いくら取れそうか
2. 今日誰に対応すべきか
3. なぜその案件が重要か
4. 何を送ればいいか
5. 実際いくら回収できたか

## 利用者タブ構成

### 1. 今日の利益

表示順：

1. 今日の回収期待値
2. CEO Daily Mission
3. 最優先案件3件
4. 危険案件
5. 返信待ち
6. AI経営ブリーフ
7. 売上予測
8. ROI

### 2. 今すぐ返信

表示順：

1. 対応案件選択
2. AI案件状況
3. AI判断
4. 返信理由
5. 返信文
6. Gmail送信
7. 実利益保存
8. ステータス変更

### 3. 顧客

表示順：

1. 顧客選択
2. 売上サマリー
3. 案件一覧
4. タイムライン
5. 返信履歴
6. メモ
7. 次アクション

### 4. 実績

表示順：

1. 今日の実利益
2. 今月の実利益
3. 累計実利益
4. 利益を生んだTOP顧客
5. 最近の成約
6. グラフ

### 5. Gmail接続

表示順：

1. Gmail接続状態
2. Gmail接続ボタン
3. メール解析
4. 返信チェック

## 管理者限定

### 開発者

販売時は非表示。

表示条件：

- dev_mode = True
- 管理者ユーザーのみ

## 削る候補

- 旧 要対応タブ
- 重複した返信画面
- 重複した実利益保存導線
- 利用者に不要な内部ログ
- RUDIA/ルディア表記

## 最終UX

社長は以下だけでよい。

1. ホームを見る
2. AIが出した案件を見る
3. 返信する
4. 実利益を保存する
5. 実績を見る
